## 開発者スポットライト #1: FujiTrace

### FujiTraceとは

FujiTraceは、日本の中小企業やフリーランスの方に向けた「おしごと AIプラットフォーム」です。

やっていることはシンプルで、**見積書・請求書・納品書・発注書・送付状といったビジネス書類の作成とAIチェック**を提供しています。フォームに必要事項を入力すると、AIが書類を自動生成し、さらに金額の計算ミスや記載漏れがないかを自動でチェックします。

### 何ができるか

現在対応しているのは以下の5種類の書類です。

- **見積書** -- 品目・数量・単価を入力すると自動計算。消費税や合計金額のミスをAIがチェック
- **請求書** -- インボイス制度対応。登録番号の記載漏れや金額の整合性を検証
- **納品書** -- 見積書・請求書との整合性チェックに対応
- **発注書** -- 仕入先への発注内容を構造化して生成
- **送付状** -- ビジネスマナーに沿った送付状を自動生成

書類の作成だけなら、正直Excelテンプレートでもできます。FujiTraceの価値は「AIによる自動チェック」にあります。たとえば、消費税の端数処理が間違っていないか、支払条件が漏れていないか、単価が業界相場から大きくかけ離れていないか。人間が目視で確認すると見落としがちなポイントを、AIが指摘します。

### 技術スタック

開発者向けメディアなので、技術的な話も少し。

- **フロントエンド**: React 18 + Vite + TailwindCSS
- **バックエンド**: Fastify 5 (TypeScript, ESM)
- **データベース**: PostgreSQL（本番） / SQLite（開発）、ORMはKnex.js
- **AI**: OpenAI API (GPT-4o-mini)
- **PDF生成**: @react-pdf/renderer（クライアントサイド生成）
- **認証**: Supabase Auth
- **デプロイ**: Vercel + Docker

AIチェック機能の裏側では、LLMによる検証とプログラマティックな算術チェックを組み合わせています。LLMだけに任せると計算ミスを見逃すことがあるので、金額の整合性チェックはNode.jsで数値を抽出して計算し直す仕組みを別途用意しています。LLMは「記載内容が妥当か」「ビジネス慣習に合っているか」の判断に使い、算術はプログラムに任せる。この二層構造が、チェック精度のカギになっています。

### なぜ作ったか

きっかけは、日本の中小企業やフリーランスの事務作業を見ていて感じた課題です。

見積書1枚作るのに30分。請求書を送る前に上司に確認してもらうのに半日待ち。納品書と請求書の金額が微妙にずれていて、取引先からやんわり指摘される。こういう小さなストレスが積み重なって、本来やるべき仕事の時間を圧迫しています。

既存のSaaSは月額数千円の固定課金が一般的ですが、月に数枚しか書類を作らないフリーランスにとっては割高です。FujiTraceは基本無料で使えて、AIチェックも5時間ごとにリセットされる無料枠で利用可能。それ以上使う方にはProプランで対応しています。「使った分だけ払う」というモデルが、小規模事業者には合っていると考えています。

### 使ってみる

FujiTraceは [fujitrace.jp](https://fujitrace.jp) で無料で使えます。アカウント登録後、ダッシュボードからすぐに書類作成を始められます。

LPにデモフォームも用意しているので、登録前に触ってみたい方はそちらからどうぞ。
