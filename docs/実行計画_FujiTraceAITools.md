## 1. プラットフォーム構想

### 1.1 ブランド

- **プラットフォーム名:** FujiTrace AI Tools
- **タグライン:** 「責任あるAI利用プラットフォーム」
- **URL構造:** `https://fujitrace.jp/tools` にプラットフォームトップ、個別ツールは `/tools/{tool-name}`
- **料金訴求:** 「初期費用0円・月額0円・AI利用量に応じた従量課金のみ」

### 1.2 設計原則（全ツール共通）

| 原則 | 内容 |
|------|------|
| ワンアカウント | 既存Supabase認証を全ツール共通で使用 |
| 共通LLMプロキシ | 全ツールが同じプロキシ経由 → 自動的にFujiTrace監視 |
| 共通課金 | 既存Stripe/プラン基盤を流用 |
| 作成＆チェック | 全ツールは「作成」→「AI自己チェック」の2段構え |
| 責任の明示 | 「これを送ると○○のリスクがある」を画面上で明示 |
| FujiTrace搭載 | 全トレースがFujiTraceダッシュボードに自動連携 |

## 3. 第1弾: AI見積書作成＆チェック

### 3.1 ゴール

全事業者が「AIに質問に答えるだけで、正確で責任ある見積書が作れる」体験を提供する。

### 3.2 ユーザーフロー

```
1. 事業情報入力（初回のみ）
   - 会社名/屋号、住所、電話、インボイス番号、振込先

2. 見積内容をAIに伝える
   - 「何の仕事？」（チャット形式でAIが質問）
   - 「単価と数量は？」
   - 「工数・納期は？」
   - 「特記事項は？」

3. AIが見積書ドラフト生成
   - 項目、数量、単価、小計、消費税、合計
   - 有効期限、納品日、支払条件

4. AI自己チェック
   - 計算ミス検出
   - 過小見積り警告（相場データと比較）
   - 消費税計算の整合性
   - 必須項目の欠落チェック
   - インボイス制度対応チェック

5. ユーザー確認・微調整

6. PDF出力 or メール送信テンプレ
```

### 3.3 責任の発生ポイント

| ミス | 実害 | AIチェックで防ぐ |
|-----|------|---------------|
| 過小見積り | 赤字受注 | 相場と比較して警告 |
| 計算ミス（小計・税・合計） | 信用失墜、修正手間 | 自動再計算で検出 |
| 消費税率の誤り | 追徴リスク | 項目別に正しい税率を適用 |
| インボイス番号欠落 | 仕入税額控除不可（相手に迷惑） | 必須項目チェック |
| 宛名の誤り | 送付ミス、失注 | 過去データと照合 |

### 3.4 差別化ポイント

| | 競合（Misoca、freee見積書等） | AI見積書作成＆チェック |
|--|-----------------------------|--------------------|
| 料金 | 月額500-2,000円 | API実費のみ（月数百円） |
| 作成方法 | フォーム入力 | **AIと対話で作成** |
| チェック機能 | なし | **AI自己チェック + 責任警告** |
| 監視 | なし | **FujiTraceで全ログ監視** |
| 対象 | 既存ユーザー | これからAI使いたい全事業者 |

### 3.5 開発スコープ（MVP）

**必須機能:**
- [ ] チャット形式の入力UI
- [ ] 業種別テンプレート（IT/建設/デザイン/コンサル/士業の5種）
- [ ] AI見積書生成ロジック
- [ ] AI自己チェックロジック
- [ ] PDF出力
- [ ] FujiTrace自動トレース連携
- [ ] 事業情報の永続化（Supabase）

**Phase 2以降:**
- 過去見積履歴
- 顧客管理
- Excel/Word出力
- 見積書→請求書の自動連携（第2弾ツール完成後）

## 4. 技術実装方針

### 4.1 アーキテクチャ

```
packages/landing/src/pages/tools/
  ├── index.tsx              # プラットフォームトップ /tools
  ├── estimate/              # AI見積書
  │   └── index.tsx          # /tools/estimate
  ├── invoice/               # AI請求書（Phase 2）
  └── email/                 # AIメール（Phase 3）

src/routes/tools/
  ├── estimate.ts            # 見積書生成API
  ├── estimate-check.ts      # チェックAPI
  └── estimate-pdf.ts        # PDF生成API

src/prompts/tools/
  └── estimate/
      ├── create.md          # 生成プロンプト
      └── check.md           # チェックプロンプト
```

### 4.2 FujiTrace連携

全ツールは内部で既存のLLMプロキシハンドラを使用する。これにより:
- 追加の統合作業なしでトレースが自動記録される
- 既存のコスト計算・PII検出・評価が全て適用される
- ワークスペース単位で利用量が集計される

### 4.3 認証・課金

- 認証: 既存Supabase認証を再利用（ログイン済みユーザーのみアクセス可）
- 無料枠: Freeプランで月10件まで
- Proプラン: 無制限 + 高度チェック機能
- ツール利用自体の課金は不要（FujiTraceの既存課金に合流）

## 6. KPI・判断基準

### 6.1 第1弾の成功基準（リリース後2週間）

| 指標 | 目標 | 判断 |
|-----|------|------|
| ユーザー登録 | 50件以上 | 未達なら訴求見直し |
| 見積書作成数 | 100件以上 | 未達ならUI改善 |
| X反応 | 100いいね以上 | 未達なら投稿内容見直し |
| 有料転換（Pro） | 1件以上 | 未達でも継続（早すぎる） |

### 6.2 ピボット判断

- 2週間で登録50未満 → LP・訴求の問題。改善して再発信
- 登録はあるが作成数が少ない → UI・フローの問題。改善
- 全く反応なし → ツール選定ミス。次のツール（AI請求書）を前倒し
