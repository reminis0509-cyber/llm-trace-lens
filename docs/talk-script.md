# FujiTrace トークスクリプト（Zoom面談用）

> 前提: AI agent企業のCTO/技術責任者とのZoom面談。
> 原則: 売り込みではなく対等な共同検証。全体像を最初から透明に提示する。

---

## Step 1: ヒアリング — 現状の課題を引き出す

```
「本日はお時間いただきありがとうございます。

早速なんですが、御社ではLLMの監視ツールは導入されておりますでしょうか？」
```

### 相手の回答パターン別の対応

**A) 「特に入れていない / 自前でログを取っている程度」**

```
「ありがとうございます。実は同じような状況の企業さんが多くて、
自前のログだと本番で問題が起きてからしか気づけないケースが結構あるんですよね。

例えば、LLMの回答品質が突然下がったとか、
特定のプロンプトだけコストが跳ね上がっているとか、
そういった問題に気づくのって今どうされていますか？」
```

**B) 「Langfuseなど海外ツールを使っている」**

```
「なるほど、Langfuseですか。使い勝手はいかがですか？

海外ツールだとUIが英語だったり、日本語のPII検出ができなかったり、
データの保存先が海外になったりで、
特にエンタープライズのお客様向けだと少し気になる点があると
聞くことがあるんですが、その辺りはいかがでしょうか？」
```

**C) 「Datadog等のAPMで見ている」**

```
「Datadogだとインフラ監視としては優秀ですけど、
LLMのプロンプトと回答の中身まで見えていますか？

例えばハルシネーションの検出とか、回答品質のスコアリングとか、
そのレベルの可視化は今どうされていますか？」
```

---

## Step 2: FujiTraceの紹介 — 無料導入の依頼

```
「ありがとうございます。

実は私たちFujiTraceという国産のLLMオブザーバビリティツールを開発しておりまして、
まさに今おっしゃったような課題を解決するために作っています。

具体的には、

- LLMへのリクエストとレスポンスを全てトレースして可視化
- レイテンシ、トークン使用量、コストをリアルタイムで監視
- 日本語のPIIを自動検出（住所、電話番号、マイナンバーなど15パターン以上）
- LLM-as-Judgeで回答品質を自動スコアリング

というものです。

導入もすごくシンプルで、既存のOpenAIやAnthropicのAPI呼び出しの
ベースURLを1行変えるだけで、コードの修正は一切不要です。

それで、お願いなんですが、
まずは無料で御社の環境に導入していただいて、
率直なフィードバックをいただけないでしょうか？

無料プランでも基本的な機能は全て使えますし、
Pro相当の機能も30日間無料でお試しいただけます。
1-2週間使っていただければ、
今まで見えていなかった問題が可視化されると思います。」
```

---

## Step 3: 全体像の透明な提示 — ルート営業の提案

> ここが最重要。隠さずに全体像を見せることで対等な関係を築く。

```
「ここからは、正直に全体像をお話しさせてください。

私たちがやりたいのは、単に御社にツールを使っていただくことだけではなくて、
御社のお客様にも価値を届けることなんです。

具体的に言うと、もし御社自身がFujiTraceを使ってみて
『これは良い』と思っていただけた場合、
御社のお客様にも無料試用をご提案いただけないかと考えています。

御社にとってのメリットは2つあります。

1つ目は、御社がお客様にAIエージェントを提案する際に、
『品質監視も込みでご提供できます』と言えること。
これは営業の説得力がかなり上がると思います。

2つ目は、御社のお客様が有料転換した際に、
マージンをお支払いします。

もちろん、これはまず御社自身が使ってみて
価値を感じていただけた場合の話です。
義務ではありません。

まずは御社の環境で試していただいて、
フィードバックをいただくところから始めませんか？」
```

## Step 4: クロージング — 次のステップの確認

```
「では、まず無料で導入していただいて、
1-2週間ほど使っていただいた後に、
もう一度お時間いただいてフィードバックをお聞きできればと思います。

導入は本当に簡単で、ベースURLを1行変えるだけです。
もし何かつまずくことがあれば、私が直接サポートしますので。

導入のタイミングなんですが、今週中とかでいけそうですか？」
```

## Step 5: フォローアップZoom — 価値実感 → ルート営業実行へ

> 1-2週間後のフォローアップZoom

**A) 価値を実感している場合**

```
「データを見ていただいて、いかがでしたか？

（相手のフィードバックを聞く）

ありがとうございます。
前回お話しした件なんですが、
御社のお客様にもPro相当30日間の無料試用をご提供できます。

御社から紹介いただいた場合、
お客様が有料プランに転換した際にマージンをお支払いします。

紹介の方法もシンプルで、
お客様にFujiTraceのURLをお伝えいただくだけです。
御社のパートナーコードを発行しますので、
そこから登録いただければ自動的に紐づきます。

まずは1社、御社の中で相性が良さそうなお客様で
試してみませんか？」
```

**B) 価値を実感していない場合**

```
「率直なご意見をありがとうございます。
具体的にどの辺りが期待と違いましたか？

（フィードバックを丁寧に聞く）

なるほど、それは改善の余地がありますね。
いただいたフィードバックを反映して、
もう一度試していただけますか？」
```

## Step 6: バンドル契約への発展

> ルート営業の実績が積み上がったタイミングで切り出す

```
「おかげさまで、御社経由で○社のお客様にご利用いただいていて、
有料転換も○社出ています。

ここまで来たら、正式にパートナー契約を結びませんか？

具体的には、御社の新規のお客様全てに
FujiTraceを標準で組み込んでいただく形です。
レベニューシェアで○○%をお支払いします。

御社にとっては、
- 全てのお客様に『品質監視込み』で提案できる
- 追加の営業コストなしでマージン収入が入る
- 御社のプロダクトの付加価値が上がる

というメリットがあります。

契約の詳細は別途お送りしますが、
方向性としていかがでしょうか？」
```

---

## 注意事項

- **売り込まない。** 常にフィードバック依頼の姿勢を崩さない
- **全体像を最初のZoomで見せる。** ルート営業の話も隠さない。隠すと営業臭がする
- **義務感を与えない。** ルート営業は「良いと思ったら」が前提
- **具体的な数字で話す。** 「品質が上がります」ではなく「15パターンのPII検出」「1行変更で導入」
- **相手の課題を先に聞く。** こちらの説明は相手の課題に合わせてカスタマイズする
