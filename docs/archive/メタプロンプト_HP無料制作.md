# メタプロンプト: HP無料制作機能の実装

> 生成日: 2026-04-06
> 最終更新: 2026-04-06（v2 — 課金フロー・認証・悪用防止のフィードバック反映）
> 用途: このプロンプトをClaude Codeの新セッションに貼り付けて、HP無料制作機能を実装させる
> 前提: `docs/戦略_2026.md` Section 11 を戦略根拠とする

---

## 目的

FujiTraceのランディングページ (`packages/landing/`) に「HP無料制作」機能を追加する。
**認証済みユーザー（無料トライアル含む）**が、業種別テンプレートを選択し事業情報を入力すると、AIチャットbot（FujiTraceウィジェット搭載済み）入りのHP一式をzipでダウンロードできる。

---

## 戦略的背景（実装者が理解すべきこと）

- これは「着火剤の着火剤」。HP制作自体で利益を取る目的ではない
- HP → チャットbot（API従量課金で回収）→ FujiTrace（将来のアップセル）のファネル
- 「LLMで価値がゼロになったサービスを無料で配り、顧客基盤を取る」モデル
- サポートは一切しない。テンプレート型・セルフサービス・ファイル渡しのみ
- **無料で配るのは「HPとチャットボットのUI」であって、「LLM API実行コスト」ではない。** ユーザーは必ずFujiTraceアカウント（トライアル or 有料プラン）を持った状態でチャットボットを利用する

---

## 前提条件

以下が既に存在・動作していることを前提とする。未実装の場合はこのメタプロンプトのスコープ外として、先に実装すること。

| 前提 | 確認方法 |
|------|---------|
| FujiTraceサインアップフロー（Supabase Auth） | `src/auth/` を確認 |
| 30日無料トライアル（Pro相当） | `src/plans/` を確認 |
| チャットbotウィジェット (`widget.js`) | `packages/landing/public/widget.js` または配信元を確認。**未実装の場合、タスク1の前にwidget.jsの実装が必要** |
| ワークスペース作成API | `src/routes/` を確認 |
| Stripe顧客登録（トライアル終了後の課金導線） | `src/billing/stripe.ts` を確認 |

---

## ユーザーフロー（課金モデルとの整合）

```
1. ユーザーが Google Ads「HP無料制作」で /hp-create に来る
2. /hp-create ページに「まずは無料アカウント作成」CTA
3. 未ログインなら /signup へリダイレクト → 30日無料トライアル開始
4. ログイン済み状態で /hp-create に戻る
5. 業種選択 → 事業情報入力 → zip生成・ダウンロード
6. ダウンロードしたHP内のチャットbotはユーザーのワークスペースに紐づく
7. 30日間はチャットbot無料で動作
8. 31日目以降: トライアル終了通知 → Proプラン移行（API従量課金開始）
```

**これにより:**
- 「無料でHP作れます」の訴求を維持
- LLM APIコストをFujiTraceが無制限に負担するリスクを排除
- ダッシュボードへのログイン導線が確保される（アップセル可能）

## 実装タスク

### タスク0: サインアップフローの確認

既存のサインアップフローが以下を満たすか確認する:

1. Supabase Auth でユーザー作成
2. デフォルトワークスペース自動作成
3. 30日Proトライアル開始
4. Stripe顧客ID紐づけ（トライアル終了後の課金準備）

**確認ファイル:** `src/auth/`, `src/routes/auth.ts`, `src/plans/`, `src/billing/stripe.ts`

上記が揃っていれば、HP制作機能はこのフローに乗るだけで良い。不足があれば先にバックエンドで対応する。

### タスク1: 業種別HPテンプレート3種の作成

以下の3業種のレスポンシブHTMLテンプレートを作成する。

**共通要件:**
- HTML5 + CSS3（TailwindCSS CDN使用可）。JavaScript最小限
- モバイルファースト・レスポンシブデザイン
- 日本語フォント: Noto Sans JP（Google Fonts CDN）
- カラーは業種に合わせたプロフェッショナルなトーン
- 以下のセクション構成を共通とする:
  1. ヘッダー（ロゴ/事業名 + ナビ）
  2. ヒーローセクション（事業名 + キャッチコピー + メイン画像）
  3. サービス紹介（3-4項目）
  4. 料金/メニュー（業種に応じて）
  5. アクセス/地図（Google Maps埋め込みプレースホルダー）
  6. お問い合わせ（電話番号 + 営業時間）
  7. フッター
- SEO最低限対応: `<title>`, `<meta description>`, OGPタグ, `lang="ja"`
- FujiTraceチャットbotウィジェットのscriptタグを `</body>` 直前に配置

**テンプレート1: 飲食店 (`restaurant`)**
- トーン: 暖色系（ダークブラウン + ゴールドアクセント）
- セクション: メニュー（カテゴリ別）、店内写真ギャラリー、予約案内
- プレースホルダー: 店名、ジャンル、住所、電話番号、営業時間、メニュー項目×6

**テンプレート2: 美容サロン (`salon`)**
- トーン: 柔らかい色調（ピンクベージュ + ホワイト）
- セクション: 施術メニュー（カット/カラー/パーマ等）、スタイリスト紹介、予約導線
- プレースホルダー: サロン名、住所、電話番号、営業時間、メニュー項目×6、スタイリスト名×2

**テンプレート3: 士業事務所 (`office`)**
- トーン: 信頼感（ネイビー + ホワイト + ゴールドアクセント）
- セクション: 取扱業務一覧、代表者プロフィール、料金目安、相談予約
- プレースホルダー: 事務所名、代表者名、資格、住所、電話番号、取扱業務×4、料金×3

**テンプレートファイルの配置:**
```
packages/landing/public/hp-templates/
  restaurant/
    index.html
    style.css
    images/         (プレースホルダー画像: Unsplash等のフリー素材)
  salon/
    index.html
    style.css
    images/
  office/
    index.html
    style.css
    images/
```

**テンプレート内のプレースホルダー記法:**
```html
{{business_name}}
{{address}}
{{phone}}
{{hours}}
{{menu_item_1_name}}
{{menu_item_1_price}}
{{chatbot_workspace_id}}
```

**画像の扱い:**
- MVPでは画像アップロード機能は実装しない（Phase 1に回す）
- テンプレート内の画像は全てプレースホルダー（フリー素材の静的ファイル）
- `<img src="images/hero.jpg" alt="{{business_name}}">` のように、altだけ置換する
- ユーザーがHP取得後、自分で画像を差し替える想定

**チャットbotウィジェット埋め込み（全テンプレート共通）:**
```html
<!-- FujiTrace AIチャットbot -->
<script
  src="https://fujitrace.jp/widget.js"
  data-workspace="{{chatbot_workspace_id}}"
  data-theme="auto"
  data-position="bottom-right"
  async
></script>
```

### タスク2: HP制作フォームページの作成

`packages/landing/src/` に `/hp-create` ページを新設する。

**認証ガード:**
- 未ログインユーザーがアクセスした場合、ページ上部に「HP無料制作を利用するには、まず無料アカウントを作成してください」バナーを表示し、「無料アカウント作成」ボタンで `/signup?redirect=/hp-create` へ遷移
- ログイン済みの場合はフォームを表示
- 認証状態の確認は既存のSupabase Auth セッションを利用

**ページ構成:**

1. **ヒーローセクション**
   - 見出し: 「AIがあなたのホームページを5分で作成。完全無料。」
   - サブテキスト: 「業種を選んで、情報を入力するだけ。AIチャットbot付きのプロ品質HPが完成します。」
   - デザイン: 既存LP（`packages/landing/src/`）のデザインシステムに準拠

2. **ステップ表示**
   - Step 1: 業種を選択
   - Step 2: 事業情報を入力
   - Step 3: ダウンロード

3. **業種選択UI**
   - 3つのカードを横並び（モバイルは縦）
   - 各カード: アイコン + 業種名 + 「このテンプレートで作成」ボタン
   - 選択するとStep 2のフォームが表示される

4. **入力フォーム（業種選択後に表示）**
   - 共通フィールド:
     - 事業名（必須）
     - 住所（必須）
     - 電話番号（必須）
     - 営業時間（任意）
     - キャッチコピー（任意、未入力の場合は業種別デフォルト）
   - 業種別フィールド:
     - 飲食: メニュー名×6 + 価格×6
     - 美容: メニュー名×6 + 価格×6 + スタイリスト名×2
     - 士業: 取扱業務×4 + 料金目安×3 + 代表者名 + 資格
   - メールアドレスはフォームに表示しない（ログイン済みセッションから取得）
   - **画像アップロードはMVPでは実装しない**

5. **生成・ダウンロードUI**
   - 「ホームページを生成する」ボタン
   - プログレスバー（生成中）
   - 「zipをダウンロード」ボタン
   - 「ダウンロード後の使い方」簡易ガイド:
     1. zipを解凍する
     2. お使いのサーバーにフォルダごとアップロードする
     3. 完了 — チャットbotは自動で動き始めます

**技術要件:**
- Reactコンポーネントとして実装（既存LPのルーティングに追加）
- フォームバリデーション: 必須フィールドのみ。電話番号の形式チェック
- zip生成: ブラウザ側で `JSZip` ライブラリを使用
  - テンプレートHTML/CSSを fetch で取得
  - プレースホルダーをフォーム入力値で置換
  - プレースホルダー画像をそのまま含める
  - zipに固めてダウンロード
- ワークスペースIDはバックエンドAPI（タスク3）から取得し、ウィジェットの `data-workspace` に注入

**重要な制約:**
- Using `any` type in TypeScript is prohibited. Use explicit types or `unknown` instead.
- 既存のLPデザインシステム（TailwindCSS、カラーパレット、フォント）に準拠すること
- 新しいnpmパッケージは `jszip` のみ許可。それ以外は要確認
- ファイルサイズ: 生成されるzipは5MB以下を目標

### タスク3: バックエンドAPI

認証済みユーザーのワークスペースIDを返し、チャットボットの初期設定を行うエンドポイント。

**エンドポイント:**
```
POST /api/hp-generate
Headers: Authorization: Bearer <supabase_token>  (認証必須)
Request Body:
{
  "business_name": "サンプル食堂",
  "template": "restaurant",
  "business_info": {
    "address": "東京都渋谷区...",
    "phone": "03-1234-5678",
    "hours": "11:00-22:00",
    "menu_items": [
      { "name": "ラーメン", "price": "900円" },
      ...
    ]
  }
}
Response:
{
  "workspace_id": "ws_xxxxx"
}
```

**処理フロー:**
1. Supabase Authトークンからユーザーを認証
2. ユーザーの既存ワークスペースIDを取得（新規作成はしない — サインアップ時に作成済み）
3. チャットボットの初期設定（以下参照）
4. ワークスペースIDを返す

**チャットボット初期設定（MVP最小限）:**
- `chatbot_settings.system_prompt` に事業基本情報を追記する:
  ```
  あなたは「{{business_name}}」のAIアシスタントです。
  住所: {{address}}
  電話番号: {{phone}}
  営業時間: {{hours}}
  お客様からの質問に丁寧にお答えください。
  詳しい情報については、お電話でお問い合わせいただくようご案内してください。
  ```
- RAGやドキュメント学習はMVPでは実装しない（Phase 1以降）
- チャットボットが事業固有の質問に答えられなくても問題ない。「詳しくはお電話でお問い合わせください」と返せばよい

**エラーハンドリング:**
- 401: 未認証 → `{ "error": "認証が必要です。ログインしてください。" }`
- 429: レートリミット超過 → `{ "error": "リクエスト制限を超えました。しばらくお待ちください。" }`
- 500: 内部エラー → `{ "error": "HP生成中にエラーが発生しました。" }`

**実装場所:** `src/routes/hp-generate.ts` を新規作成し、`src/server.ts` に登録

### タスク4: 悪用防止

無料HP生成の悪用を防ぐミドルウェアを実装する。

| 対策 | 内容 |
|------|------|
| レートリミット | 同一ユーザーからのHP生成リクエストは1時間に3回まで |
| ワークスペース制限 | 1アカウントにつきHP生成は10回まで（生成回数をDBに記録） |
| 認証必須 | 未認証リクエストは一律拒否（タスク3のエンドポイントで実装済み） |

**実装方法:**
- レートリミットは既存のミドルウェアパターン（`src/middleware/`）に準拠
- 生成回数カウントは `hp_generation_count` フィールドをユーザーテーブルに追加（またはワークスペーステーブル）

## 実装しないこと（スコープ外 — Phase 1以降）

- 画像アップロード機能（MVPではプレースホルダー画像のみ）
- HP編集機能（ダウンロード後の変更はユーザー責任）
- ホスティングサービス（ファイル渡しのみ）
- カスタムドメイン対応
- テンプレート4種目以降（まず3種で検証）
- チャットbotのRAG/ドキュメント学習（MVPでは固定プロンプトのみ）
- reCAPTCHA（認証必須なので優先度低。スパムが問題になったら追加）

## 受け入れ基準

1. 未ログインで `/hp-create` にアクセスすると、サインアップ誘導バナーが表示される
2. ログイン済みで `/hp-create` にアクセスすると、業種選択フォームが表示される
3. 業種を選択し情報を入力して「生成」を押すと、zipがダウンロードされる
4. ダウンロードしたzipを解凍し、`index.html` をブラウザで開くと、入力した情報が反映されたHPが表示される
5. HP内にFujiTraceチャットbotウィジェットが表示され、クリックするとチャット画面が開く（**widget.jsが実装済みの場合**）
6. 3業種すべてのテンプレートが機能する
7. モバイルで `/hp-create` ページおよび生成されたHPが正常に表示される
8. フォーム未入力の必須項目にはバリデーションエラーが表示される
9. 1時間に4回目のHP生成をしようとすると、レートリミットエラーが表示される
