本ドキュメントは FujiTrace オンボーディングチュートリアル Step 2 で再生する 60 秒デモ動画の絵コンテである。Benesse 型 2 段階成長モデルにおける「プレビュー型チュートリアル」の中核資産であり、Free/Pro ユーザーに Pro+「AI 事務員」プランへの欲求を喚起することを目的とする。

## 2. デモで見せる具体タスク

**シナリオ名**: 月末の請求書一括発行

**ユーザー入力（チャット欄 1 行）**:

> 今月分の請求書をまとめて作成して。取引先は先月と同じ5社で。

**エージェント挙動（5 ステップ）**:

1. 先月の請求書履歴を検索（`invoice.search_history`）
2. 5 社分の請求書を順次生成（`accounting.invoice_create` × 5）
3. 金額・日付の算術チェック（`arithmetic.check`）
4. ユーザーに承認を求める（approval gate）
5. 承認後に PDF を一括ダウンロード可能な状態へ

**このシナリオを選んだ理由**:

- 複数ツールの組み合わせが自然に発生し、Tool Trajectory の価値が可視化できる
- 月末請求業務は日本企業の業務リアリティに合致し、Pro 層の想定ユーザー（個人事業主から小規模法人）に刺さる
- approval gate を差し込めるため「最終確認は人間」という責任 AI ツール戦略との整合を演出で担保できる
- trace の多様化（履歴検索→生成→検算）が 60 秒で成立する複雑度

## 5. 画面表示要素の設計

### 5.1 チャット入力エリア

- 位置: ダッシュボード中央下部、センター配置（2026-04-11 リデザイン後の AI 事務員チャット UI を踏襲）
- プレースホルダ: 「やってほしいことを一文で入力」
- 入力中は下部にサジェストチップ 4 種を薄く表示（「請求書を作成」「見積書をチェック」「納品書を生成」「送付状を作成」）。ただしデモ中はユーザーが直接タイプするため未選択
- フォント: Noto Sans JP、16px、黒 #111

### 5.2 スケルトン trace パネル

既存 `StreamingSkeletonTrace` コンポーネント（`skeleton_trace_feature.md` 準拠）を参考にした 3 状態表示:

- Pending（未着手）: 灰色・半透明のチェーン行
- In Progress（処理中）: 青いパルス・経過秒数「○s」を右端に表示
- Complete（完了）: 緑のチェックアイコン・最終所要時間

階層構造:

```
計画 [done, 1s]
 ├─ invoice.search_history [done, 2s]
 ├─ accounting.invoice_create × 5 [done, 6s]
 │   ├─ 株式会社A [done]
 │   ├─ 株式会社B [done]
 │   ├─ 合同会社C [done]
 │   ├─ D 商店 [done]
 │   └─ 株式会社E [done]
 ├─ arithmetic.check [done, 3s]
 └─ approval_gate [waiting → approved, 4s]
```

### 5.3 Tool Trajectory 可視化

右サイドパネルに時系列バー（ガントチャート風、横軸が秒）。各ツールが色分け:

- `invoice.search_history`: 青
- `accounting.invoice_create`: 紫（5 本並列）
- `arithmetic.check`: 緑
- `approval_gate`: 橙

バーの左端に開始時刻、右端に完了時刻、hover 時にログが見える風のモックアップを想定（動画内では hover 演出なし、静的表示）。

### 5.4 承認ゲート UI

モーダルダイアログ（中央オーバーレイ）:

- タイトル: 「この内容で発行してよろしいですか？」
- サブテキスト: 「AI は手順を実行しました。最終承認をお願いします。」
- 本文: 5 件の請求書サムネイル（横並びグリッド、各カードに取引先名・金額・発行日）
- 下部 2 ボタン: 左「修正を依頼」（グレー）、右「承認して発行」（Primary Blue #2563eb）
- ダイアログの周囲は半透明黒オーバーレイで背景がわずかに見える

### 5.5 最終結果プレビュー

5 枚の請求書 PDF サムネイル（縦長 A4 比率）をグリッド（3 + 2 配置）で整列。各サムネイルに:

- 右下: ダウンロードアイコン
- 左上: 取引先名ラベル
- 中央: 生成された PDF の 1 ページ目縮小画像（金額行が読める解像度）

### 5.6 CTA カード

全画面白背景にフェード遷移。中央に:

- 上段: FujiTrace ロゴ（既存 `docs/assets/fujitrace-logo-light-1024.png`）
- 中段: 大文字「Pro+『AI 事務員』でご利用いただけます」
- 下段: 小さく「月額 ¥9,800 / 1 名から」（料金表記は Founder 確認後に追加/削除判断）

## 6. 制作方法の推奨

### 案 A: 画面収録ベース

- 実施タイミング: Phase A1（2026-06-07 予定）の自律エージェント MVP 完成後
- ツール: macOS QuickTime または OBS Studio で画面収録、DaVinci Resolve で編集
- メリット: 実動作そのままのため信頼性が高い、制作コスト低
- デメリット: Phase A1 実装完了を待つ必要がある、UI 変更のたび再撮影が必要、60 秒に収めるために業務ロジックの待ち時間を早送りする編集が必要

### 案 B: モーショングラフィックス

- 実施タイミング: 即時着手可能
- ツール: Figma でフレーム制作 → After Effects または Rive でアニメーション化
- メリット: Phase A1 を待たずに制作開始できる、尺と演出の完全コントロールが可能、再撮影コストが低い
- デメリット: 「実際の動作と違う」と誤解されるリスク、制作コスト高（外注で 30〜60 万円想定）、Phase A1 の UI と乖離するリスク

### 推奨: ハイブリッド（初版 B → 本版 A 差し替え）

- 初版 v1（2026-04 末〜 5 月）: 案 B で制作し、LP ヒーローとチュートリアルに即時投入
- 本版 v2（Phase A1 完成後、2026-06 以降）: 案 A で差し替え、誇大表現リスクを除去
- 差し替え時は URL と埋め込み ID を維持して運用影響を最小化

### Founder 判断が必要な事項

- 初版 v1 を外注するか内製するか（内製の場合 Figma + Rive で Founder 自身が作業可能か）
- 制作予算の上限（外注時）
- CTA 文言に料金（¥9,800）を入れるか、Pro+ への誘導のみに留めるか
- 初版 v1 リリース予定日（GTM プッシュ施策との整合）

## 7. BGM / 効果音の方向性

### BGM 候補（無料素材、商用利用可）

1. **DOVA-SYNDROME** — カテゴリ「企業・コーポレート」の ambient 系。日本サイトで利用規約が明確
2. **YouTube Audio Library** — 「Ambient / Calm」タグの 60 秒以上の楽曲
3. **甘茶の音楽工房** — 「環境音楽」カテゴリ、法人利用実績多数

選定基準: BPM 60〜80、メロディが前に出ない、ループ耐性あり、日本企業向け SaaS 動画で使用実績がある音源を優先。

### 効果音設計

既存の `skeleton_trace_feature.md` の Web Audio API 設計と整合させる:

- **ツール呼び出し音**: sine 波 400Hz、200ms、Attack 10ms / Decay 190ms（既存 StreamingSkeletonTrace の音と揃える）
- **検算完了音**: sine 波 800Hz、150ms、軽いリバーブ
- **承認通知音**: 2 音の上行（C5 → E5）、各 100ms
- **完了音**: 3 音の上行（C5 → E5 → G5）、各 80ms
- **キータイプ音**: 既製のメカニカルキーボード系音源、-18dB まで絞る

### 音声 OFF 成立要件

- 全ての情報はハードサブで提示
- 効果音が伝える情報（ツール呼び出し、完了）は画面上のアイコン遷移でも同時に表現
- BGM と効果音は「雰囲気の補強」に留め、BGM が無くても動画の意味が変わらないこと

## 8. 検証チェックリスト

制作完了後、公開前に以下を全てチェックする。

- [ ] 60 秒以内に収まっている（59〜60 秒）
- [ ] 音声 OFF で視聴しても意味が完全に伝わる
- [ ] CTA が明確（「Pro+『AI 事務員』でご利用いただけます」が最後 5 秒に存在）
- [ ] 実装と乖離しない範囲の演出（誇大広告にならない）
- [ ] 承認ゲートの存在が明示されている（approval gate カットが 5 秒以上）
- [ ] 絵文字なし、派手な演出なし（パーティクル、カメラシェイク、過度なズーム禁止）
- [ ] ダッシュボード埋め込み（幅 960px 想定）と LP ヒーロー（幅 1280px 想定）の両方で再生可能な解像度（1920x1080 マスターから両用にトランスコード）
- [ ] 責任 AI ツール戦略と矛盾しない（自律 = scaffold を外すと解釈されない。ツール呼び出しが明示的に trace される、人間承認が中心にある）
- [ ] 競合プロダクト名（freee、Claude Code、Operator、Devin、Manus 等）が画面にもナレーションにも含まれない
- [ ] 固有名詞の誤字なし（FujiTrace、Pro+、AI 事務員の表記ゆれ無し）
- [ ] ハードサブのフォントが Noto Sans JP、サイズは最低 24px（ダッシュボード埋込時に縮小されても読める）
- [ ] 字幕の背景帯またはアウトラインで可読性を担保（白背景に字が乗るカットで字が消えないこと）
- [ ] 最終 5 秒の CTA カードが単体でスクリーンショットとして成立する（LP の OGP 代替としても使えること）
